关于「您的连接不是私密连接」与 ERR_CERT_COMMON_NAME_INVALID
================================================================

一、常见原因（在国内尤其多见）
----------------------------
1. 部分宽带/公司网/公共 WiFi 会对境外网站做 HTTPS 拦截，浏览器收到的证书
   不是 Vercel 颁发的，就会报「证书名称无效」。
2. 手机系统时间不准，也可能导致证书校验失败（较少见）。
3. 安装了「安全软件 / 抓包工具」对 HTTPS 解密，证书会不匹配。

二、您可以先自行排查
----------------------------
• 换网络试：例如关掉 WiFi，改用 4G/5G；或换一个路由器/热点。
• 确认地址完整：https://songhe-online.vercel.app（不要改成 http、不要少字母）。
• 核对手机/电脑「日期与时间」是否为自动、是否准确。
• 暂时关闭杀毒/管家类软件的「HTTPS 扫描」再试。

三、若 Vercel 在您所在网络长期不可用
----------------------------
同一套静态文件可部署到国内更易打开的地址，例如：

【推荐】Gitee Pages（国内访问通常更顺）
1. 注册 gitee.com，新建公开仓库。
2. 把本目录下这些文件提交到仓库根目录（或 docs 目录，按 Gitee 设置）：
   index.html、songhe-built.css，以及 BootCDN 的脚本无需改。
3. 仓库 → 服务 → Gitee Pages → 开启，会得到 https://你的用户名.gitee.io/仓库名/

【备选】Netlify Drop
打开 https://app.netlify.com/drop ，拖入本文件夹，得到 https://xxx.netlify.app
（仍属境外，但部分网络下与 Vercel 表现可能不同。）

四、更新页面后
----------------------------
在本目录执行：npm run build:css
再把新的 index.html 与 songhe-built.css 上传到 Gitee 或重新拖 Netlify。
